# 附身规则定稿

## 1. 核心模型

- 世界中始终只存在一个活跃操控对象。
- 附身参考旁观者模式的相机绑定思路，但不是简单观战。
- 被附身生物作为临时载体保留在世界中，拥有独立存在。
- 附身期间，玩家的控制优先级高于目标实体原本行为。

## 2. 生命值规则

- 附身时，目标生物获得临时增益。
- 附身状态下，玩家当前生命值以“增益后的目标生命值”为准。
- 玩家与被附身生物的生命值彼此独立。
- 附身开始时锁定一次结算基准值，后续不再随目标临时血量变化而重算。
- 被附身生物死亡时：
  - 若其生命值不高于玩家，则玩家直接脱离，生命值不受影响。
  - 若其生命值高于玩家，则玩家失去 `x` 点生命值。
  - `x = 被附身生物总生命值 - 玩家生命值`
  - `x` 需要做上限限制，且玩家最低保留半颗心。
- 主动退出时：
  - 玩家生命值保持附身前状态不变。
- 因目标死亡被动退出时：
  - 玩家最终生命值按 `y = 附身前生命值 - x` 结算。
  - `y` 最低保留半颗心，不允许降到 0 或负数。

## 3. 状态生命周期

- 附身开始时记录玩家原始生命值。
- 附身结束时区分两种退出：
  - 主动退出
  - 目标死亡导致的被动退出
- 退出后恢复原本相机和控制权。
- 目标实体继续按原版逻辑独立存在，除非它本身死亡或被其他机制移除。

## 4. 控制优先级

建议采用以下优先级：

1. 玩家附身输入
2. 当前载体的可执行原版能力
3. 原版实体默认 AI
4. 原版通用物理和伤害逻辑

原则：

- 能由玩家控制的动作优先由玩家输入驱动。
- 玩家不能执行的动作，回退到实体本身的原版限制。
- 不强行让所有实体共享同一套控制模板。
- 附身期间目标实体的原版 AI 暂停，只保留物理、受伤、死亡和世界交互等原版事件。

## 5. 原版适配原则

- 尽量复用原版实体、伤害、传送、死亡、维度切换逻辑。
- 只在必要时使用 Mixin 或事件注入。
- 不替换底层玩家生命周期。
- 不破坏多人联机同步。
- 不改写原版 AI 体系，只在附身期间临时压制或替换控制来源。
- 目标实体死亡流程保持原版执行，掉落、经验、仇恨和死亡效果不额外改写。

## 6. 技术实现方向

- 客户端：
  - 按键触发附身和退出
  - 切换相机实体
  - 显示 HUD
- 服务端：
  - 校验附身请求
  - 维护附身会话
  - 处理增益和血量结算
  - 自动回收状态
- 网络：
  - C2S 请求附身
  - C2S 请求退出
  - S2C 批准、拒绝和强制退出

## 7. 第一版最小目标

- 对准实体请求附身
- 切换到目标视角
- 可主动退出
- 目标死亡自动脱离
- 附身期间有临时增益
- 生命值按规则结算
